## 2. 算法设计现状记录

### 2.1 Panopticon（`panopticon/`）
- 代码定位：基于 DINOv2 架构扩展，核心创新集中在：
  - `panopticon/dinov2/models/panopticon.py`
  - `panopticon/dinov2/data/augmentations.py`
- 当前设计重点（依据 `panopticon/README_ch.md`）：
  - 面向“任意传感器”的统一表示学习；
  - 输入采用字典形式（图像 + 通道 ID），支持可变通道建模；
  - 对 SAR 输入可通过 `chn_ids` 机制编码传感器/通道属性；
  - 支持抽取图像级 token 与 patch 级 token，适合分类与分割/变化检测下游任务。
- 与本项目的关联：
  - 可作为多通道 SAR（强度+相干）双时相编码 backbone；
  - 可为后续 `PanoAdaptFlood` 方案提供可变通道表示能力。

### 2.2 AdaptOVCD（`AdaptOVCD-main/`）
- 代码定位：训练自由（training-free）的开放词汇变化检测框架。
- 主要入口：
  - `AdaptOVCD-main/evaluate.py`（二分类 CD 评估）
  - `AdaptOVCD-main/evaluate_second.py`（SECOND 多类别评估）
  - `AdaptOVCD-main/changeformer/`（比较、识别、阈值等核心流程）
  - `AdaptOVCD-main/Module/`（增强模块，如自适应阈值等）
- 当前设计重点（依据 `AdaptOVCD-main/README.md`）：
  - 使用多基础模型协同，不做任务特定训练；
  - 通过自然语言定义变化类别，实现零样本变化检测；
  - 数据侧强调统一目录范式（A/B/label 或 SECOND 专用结构），并提供数据预处理脚本。
- 与本项目的关联：
  - 提供标准 CD 数据组织约定（A/B/label）与评估入口；
  - 可作为后续 UrbanSARFloods 数据接入与对比评测基线框架。


## 3. UrbanSARFloods 数据处理程序构建记录

### 3.1 需求与输入数据
- 原始数据目录：`datasets/urban_sar_floods`
- 原始结构：
  - `01_NF/`, `02_FO/`, `03_FU/`
  - `Train_dataset.txt`, `Valid_dataset.txt`, `data_norm.txt`
- 原始样本格式：
  - SAR：单文件 8 通道 GeoTIFF（float32）
  - GT：单通道 GeoTIFF
- 任务要求：
  - 将“单文件多通道”解耦为标准 CD 的 `T1/T2/Label`；
  - 采用可直接对接通用 CD 训练代码的目录结构。

### 3.2 脚本实现
- 脚本路径：`datasets/script/convert_urban_sar_floods_to_cd.py`
- 输出路径（本次实际执行）：`datasets/urban_sar_floods_CD`
- 输出结构：
  - `train/A_tif`, `train/B_tif`, `train/A_npy`, `train/B_npy`, `train/label`
  - `val/A_tif`, `val/B_tif`, `val/A_npy`, `val/B_npy`, `val/label`
  - `meta/train_list.txt`, `meta/val_list.txt`, `meta/channel_map.json`, `meta/convert_report.json`

### 3.3 通道解耦规则
- 输入 8 通道（1-based）：
  1 pre_intensity_vv
  2 pre_intensity_vh
  3 post_intensity_vv
  4 post_intensity_vh
  5 pre_coherence_vv
  6 pre_coherence_vh
  7 post_coherence_vv
  8 post_coherence_vh
- 输出：
  - `A`（T1，4通道）= `[1,2,5,6]`
  - `B`（T2，4通道）= `[3,4,7,8]`
  - `label` 保留三分类值 `0/1/2`（uint8）

### 3.4 程序参数与行为
- 关键参数：
  - `--src-root`, `--out-root`
  - `--train-list`, `--val-list`
  - `--overwrite`
  - `--strict / --no-strict`
  - `--dry-run`
  - `--workers`
  - `--max-samples`
- 行为说明：
  - `--overwrite`：覆盖已有输出；
  - `--strict`：遇到首个错误即停止；
  - `--dry-run`：仅做全流程检查，不写输出文件。
