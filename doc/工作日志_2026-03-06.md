# 工作日志（2026-03-06）

## 1. 本次目标
- 在 `UrbanSARFloods` 数据处理链路中补齐两项能力：
  - `512 -> 256` 切片并扩展输入通道 `8 -> 12`；
  - 基于原始 `GT(0/1/2)` 重构两套层次化监督标签。

## 2. 新增/落地脚本
- `datasets/script/tile_urban_sar_floods_to_256_ch12.py`
  - 输入：`datasets/urban_sar_floods`
  - 输出：`datasets/urban_sar_floods_ch12_256`
  - 关键逻辑：
    - 切片大小：`256`
    - 保持目录与 split 文件结构不变
    - 原始 8 通道扩展为 12 通道（新增差分）：
      - `dcoh_vh = B1 - B3`
      - `dcoh_vv = B2 - B4`
      - `dint_vh = B7 - B5`
      - `dint_vv = B8 - B6`
    - 过滤规则：
      - `GT` 全 0 切片丢弃
      - 原始 8 通道中任一通道全 NaN 切片丢弃

- `datasets/script/rebuild_urban_sar_floods_hier_labels.py`
  - 作用：在 `datasets/urban_sar_floods_ch12_256` 上重构层次化标签
  - 输入标签语义：`0=BG, 1=FO, 2=FU`
  - 输出目录（并行，不覆盖原 `GT`）：
    - `GT_floodness/`
    - `GT_flood_type/`
  - 标签映射：
    - Floodness：`BG->0, FO/FU->1`
    - Flood-Type：`FO->0, FU->1, BG->255(ignore)`

## 3. 关键命令
```bash
# 12 通道切片生成
python datasets/script/tile_urban_sar_floods_to_256_ch12.py \
  --src-root datasets/urban_sar_floods \
  --out-root datasets/urban_sar_floods_ch12_256 \
  --overwrite --strict

# 层次化标签重构
python datasets/script/rebuild_urban_sar_floods_hier_labels.py \
  --data-root datasets/urban_sar_floods_ch12_256 \
  --ignore-index 255 \
  --overwrite --strict
```
